Product Requirements Document (PRD)
Product Name

KMZ Waypoint Flight Planner for DJI Mini 4 Pro

1. Purpose & Vision

The purpose of this application is to enable users to:

Plan autonomous waypoint flights on a map

Configure flight and camera parameters

Export the flight plan as a DJI-compatible KMZ file

Use the KMZ file with the DJI Mini 4 Pro via the DJI Fly app workaround (file replacement)

The application does not communicate directly with the drone.
It focuses on offline mission planning and KMZ generation.

2. Target Users

DJI Mini 4 Pro pilots

Mapping / inspection / repeatable flight users

Users frustrated by DJI Fly’s limited waypoint editing & lack of import

Semi-technical users willing to copy files manually

3. Supported Platform (MVP)

Web Application (Desktop-first)

Optional future expansion: Desktop app (Electron) or Mobile

4. Functional Requirements
4.1 Map & Area Selection

The application SHALL:

Display an interactive map (Google Maps, Mapbox, or OpenStreetMap)

Allow users to:

Search by address or GPS coordinates

Zoom, pan, and rotate the map

4.2 Waypoint Creation

The application SHALL support two mission types:

A) Polygon / Area Mapping Mission

User draws a polygon (3+ points)

System automatically generates waypoints in a grid pattern

Configurable overlap and spacing

B) Manual Waypoint Mission

User places individual waypoints by clicking on the map

Each waypoint is editable independently

4.3 Waypoint Parameters

Each waypoint SHALL support:

Altitude (relative to takeoff)

Heading / yaw

Gimbal pitch

Speed (optional, global or per waypoint)

Action type:

No action

Take photo

Timed photo interval

4.4 Global Mission Settings

The mission SHALL allow configuration of:

Default altitude

Flight speed

Photo interval (seconds or distance-based)

Finish action:

Hover

Return to Home

Land

5. KMZ File Generation
5.1 Export Format

The application SHALL export missions as:

.kmz (compressed KML)

The KMZ MUST:

Follow DJI Waypoint Flight schema used by DJI Fly

Be readable by DJI Fly when placed into an existing waypoint mission folder

5.2 DJI Compatibility Constraints (Critical)

The application MUST respect:

DJI Mini 4 Pro waypoint limitations

No unsupported actions (e.g. complex SDK-only commands)

Valid coordinate ordering

Valid altitude ranges

Valid camera command syntax

Non-compliant KMZ files MUST be rejected before export.

6. DJI Fly Integration Workflow (Non-Programmatic)

Because DJI Fly has no official import feature, the app SHALL assume this workflow:

User creates a dummy waypoint mission in DJI Fly

DJI Fly generates a mission folder and KMZ file internally

User replaces the original KMZ file with the one generated by this app

The filename of the KMZ MUST match exactly

7. User Guidance Features (Required)

The app SHALL provide:

Clear instructions explaining:

Where DJI Fly stores waypoint missions

How to replace the KMZ file

File naming requirements

Warnings when user settings exceed DJI limits

A “DJI Mini 4 Pro Compatibility Check” before export

8. File & Data Structure
8.1 Internal Mission Model

Each mission SHALL contain:

Mission metadata (name, creation date)

Waypoint array

Global settings

Export version

8.2 KMZ Structure (Abstract)
mission.kmz
 └── doc.kml
     ├── MissionConfig
     ├── WaypointList
     │   ├── Waypoint 1
     │   ├── Waypoint 2
     │   └── ...

9. Non-Functional Requirements
Performance

Map interaction must feel real-time

KMZ export < 1 second for missions up to 500 waypoints

Reliability

No corrupted KMZ files

Validation before export is mandatory

Usability

No DJI account required

No drone connection required

Desktop mouse-first UI

10. Out of Scope (MVP)

Direct drone control

Live telemetry

DJI SDK integration

Automatic upload to DJI Fly

Cloud sync (optional later)

11. Risks & Constraints

DJI may change KMZ schema in future firmware

DJI Fly updates could break compatibility

No official DJI documentation for waypoint KMZ format

Reverse-engineered behavior must be tested carefully

12. Success Criteria

The product is successful if:

A user can plan a flight entirely outside DJI Fly

Export a KMZ file

Replace a DJI Fly mission KMZ

Fly the mission successfully on a DJI Mini 4 Pro without errors

13. Future Enhancements (Optional)

Terrain awareness

Elevation-based altitude adjustment

Mission versioning

Batch export

Litchi-compatible formats

3D preview of flight path
